# ARCHITECTURE.md — System Architecture

> Generated by codemap | Last updated: 2026-07-02

## Pattern: Extension Module (pi-specific)

The project follows pi's **Extension Module** pattern — a single default export function receiving `ExtensionAPI` that registers a model provider, OAuth hooks, and error handlers with the pi runtime.

## Entry Point

`src/index.ts` exports a single async default function:
```typescript
export default async function (pi: ExtensionAPI)
```

This function:
1. Resolves the API base URL (`resolveApiBase()`)
2. Resolves the API key (`resolveApiKey()`)
3. Discovers models (`resolveModels()` — dynamic first, static fallback)
4. Registers the `"agy"` provider via `pi.registerProvider()`
5. Subscribes to `"message_end"` for error surfacing

## Dependency Graph

```
src/index.ts (entry)
  ├── src/env.ts          — constants, env helpers, URL builder, key sanitization
  ├── src/auth.ts         — API key resolution chain
  │   └── src/config-store.ts — credential store traversal
  │       └── src/utils.ts     — type guards (isRecord, stringValue, etc.)
  ├── src/models.ts       — static model catalog + dynamic model discovery
  │   └── src/utils.ts
  ├── src/oauth.ts        — /login flow (agy OAuth reuse + manual paste)
  │   └── src/config-store.ts
  └── src/error-handler.ts — message_end event pipeline
      └── src/errors.ts    — classifyGeminiError()
```

## Module Responsibilities

### `src/index.ts` — Extension Entry
**Role**: Orchestrator. Wires everything together and registers with pi.
- No business logic — pure wiring.
- Handles model input array cloning for immutability.

### `src/models.ts` — Model Catalog & Discovery
**Role**: Defines available models and fetches remote model list.
- **Static catalog**: 2 hardcoded `ModelConfig` objects (Gemini 3.5 Flash, Gemini 3.1 Pro).
- **Dynamic discovery**: Fetches `/models` from the Gemini API at registration time.
- **Fallback**: On any error (network, 404, parse), silently returns static catalog.
- **Thinking level maps**: Explicit 6-level matrices (`off` through `xhigh`) mapping to `reasoning_effort` strings.

### `src/auth.ts` — API Key Resolution
**Role**: Resolve credentials from multiple sources in priority order.
- Pure function — all I/O injectable via `AuthKeyOptions`.
- Priority chain: provided key → `GEMINI_API_KEY` → `GOOGLE_API_KEY` → agy OAuth → pi `auth.json`.

### `src/config-store.ts` — Credential Store Traversal
**Role**: Shared infrastructure for walking credential files.
- `defaultAuthPaths()`: Returns ordered list of 3 file paths.
- `walkAuthPaths()`: Generic iterator over auth files with ENOENT suppression.
- `resolveAgyOAuthToken()`: Extract OAuth token from agy's file formats.

### `src/oauth.ts` — pi /login Integration
**Role**: Implements pi's OAuth callbacks (`login`, `refreshToken`, `getApiKey`).
- **Auto-login**: If agy credentials exist, logs in instantly.
- **Manual fallback**: Opens AI Studio URL, prompts user to paste API key.
- **Token lifecycle**: API keys set ~10 year expiry; agy OAuth tokens set ~55 min expiry.

### `src/env.ts` — Constants & Environment
**Role**: Single source of truth for constants, env var names, and URL construction.
- `resolveApiBase()`: Override detection with normalization.
- `sanitizeApiKey()`: Strip terminal paste wrappers and control characters.
- `buildEndpointUrl()`: Construct full endpoint URLs.

### `src/errors.ts` — Error Classification
**Role**: Map raw Gemini error messages to user-friendly, actionable messages.
- 4 error types: `invalid_key`, `rate_limited`, `quota_exceeded`, `unknown`.
- Case-insensitive keyword matching against error body.
- Each type has a full user-facing message with recovery instructions.

### `src/error-handler.ts` — Error Surface Pipeline
**Role**: Pipeline: **Filter → Classify → Deliver**.
- **Filter**: Only Gemini errors (`stopReason="error"`, provider matches `"agy"`).
- **Classify**: Delegates to `classifyGeminiError()`.
- **Deliver**: `ctx.ui.notify()` in UI mode, `console.error()` in TUI mode.

### `src/utils.ts` — Type Guards
**Role**: Shared runtime type validation for I/O boundaries.
- `isRecord(value)`: Plain object check (rejects arrays, null).
- `stringValue(value)` / `numberValue(value)` / `booleanValue(value)`: Narrowed extractors.

## Data Flow

```
1. pi starts
   ↓
2. Extension function invoked with ExtensionAPI
   ↓
3. resolveApiBase() → base URL (env or default)
   ↓
4. resolveApiKey() → credentials (env → files → pi auth)
   ↓
5. fetchRemoteModels() → live model list (or fall back to static)
   ↓
6. pi.registerProvider("agy", {…}) → provider registered
   ↓
7. pi.on("message_end", handleGeminiError) → error handler wired
   ↓
8. User chat messages flow through pi's openai-completions streaming
   ↓
9. On error, message_end event → handleGeminiError → classify → notify
```

## Key Abstractions

### Injection-Testable I/O

All I/O operations are injectable via options objects, enabling unit testing without mocking globals:

```typescript
interface AuthKeyOptions {
  env?: Record<string, string | undefined>;
  authPaths?: readonly string[];
  homeDir?: () => string;
  readFile?: (path: string) => string;
  fileExists?: (path: string) => boolean;
}

interface RemoteModelsOptions {
  apiBase?: string;
  apiKey?: string;
  fetch?: typeof globalThis.fetch;
  timeoutMs?: number;
}
```

### Thinking Level Maps

Every model explicitly declares all 6 thinking levels (`off` through `xhigh`), each mapped to a provider-specific `reasoning_effort` string or `null` (unsupported). There are no implicit defaults at the model level (though `DEFAULT_THINKING_LEVEL_MAP` serves as a fallback for remote models without static matches).

### Graceful Degradation

Dynamic model discovery, credential file reading, and API calls all use try/catch with silent fallback — the extension never crashes pi on startup.
