# CONVENTIONS.md ‚Äî Coding Conventions

> Generated by codemap | Last updated: 2026-07-02

## TypeScript

### Strict Mode

- `strict: true` in `tsconfig.json`
- No implicit `any`
- `unknown` at all I/O boundaries (API responses, file reads, env vars)
- Run-time type guards before accessing unknown data

### Module System

- ESM only (`"type": "module"`)
- `.js` extension in imports (TypeScript resolution)
- No `import * as` for local modules ‚Äî prefer named imports

### Style

- **Formatting**: oxfmt (Rust-based), no configuration
- **Linting**: oxlint with plugins: `typescript`, `unicorn`, `oxc`, `import`, `jest`
- **File size**: Modules under 300 lines
- **Exports**: All public exports have JSDoc block comments

## Patterns

### Dependency Injection for I/O

All functions that touch filesystem, environment, or network accept an injectable options object. Example:

```typescript
// From src/config-store.ts
export function walkAuthPaths<T>(
  options: AuthKeyOptions,
  extract: (parsed: Record<string, unknown> | string, path: string) => T | undefined,
): T | undefined {
  const home = options.homeDir?.() ?? homedir();
  const readFile = options.readFile ?? ((p: string) => readFileSync(p, "utf-8"));
  const fileExists = options.fileExists ?? ((p: string) => existsSync(p));
  // ...
}
```

Production defaults use real `fs`, `os`, `process.env`. Tests inject mocks.

### Type Guards at I/O Boundaries

```typescript
// From src/utils.ts
export function isRecord(value: unknown): value is Record<string, unknown> {
  return typeof value === "object" && value !== null && !Array.isArray(value);
}

export function stringValue(value: unknown): string | undefined {
  return typeof value === "string" ? value : undefined;
}
```

Used extensively in `models.ts` (parsing API responses) and `config-store.ts` (parsing JSON files).

### Explicit Error Classification

Errors are mapped to specific types with full user-facing messages that include recovery steps:

```typescript
// From src/errors.ts
export const GEMINI_ERROR_MESSAGES: Record<GeminiErrorType, string> = {
  invalid_key: "Gemini API key is invalid or expired. Run `pi /login`...",
  rate_limited: "Gemini rate limit reached. Wait a moment and try again...",
  quota_exceeded: "Gemini API quota exceeded. Check your usage...",
  unknown: "Gemini request failed. Check your API key...",
};
```

### Graceful Degradation

Every external call degrades silently:
- **Model discovery**: `fetchRemoteModels()` returns `undefined` on error ‚Üí static fallback
- **Credential reading**: `walkAuthPaths()` suppresses ENOENT ‚Üí next file in chain
- **Error handling**: Non-Gemini errors silently pass through

### Immutable Data

Model input arrays are cloned before registration:
```typescript
// From src/index.ts
models: models.map((model) => ({
  ...model,
  input: [...model.input],
}))
```

## Naming

### Functions
- `resolve*` ‚Äî resolve a value from multiple sources with fallback (e.g., `resolveApiKey`, `resolveApiBase`, `resolveModels`)
- `fetch*` ‚Äî async I/O operations (e.g., `fetchRemoteModels`)
- `walk*` ‚Äî iterate over a collection, possibly with early exit (e.g., `walkAuthPaths`)
- `build*` ‚Äî construct a value from components (e.g., `buildEndpointUrl`)
- `classify*` ‚Äî categorize input into discrete types (e.g., `classifyGeminiError`)
- `sanitize*` ‚Äî clean untrusted input (e.g., `sanitizeApiKey`)
- `handle*` ‚Äî event handler (e.g., `handleGeminiError`)

### Types
- PascalCase: `ModelConfig`, `ThinkingLevelMap`, `AuthKeyOptions`, `RemoteModelsOptions`, `RawModelEntry`
- `*Options` suffix for dependency-injected config objects
- `*Map` suffix for record/constant lookup tables

### Constants
- UPPER_SNAKE_CASE for module-level constants: `MODELS`, `DEFAULT_API_BASE`, `ENV_API_KEY`
- Prefix `ENV_` for environment variable names: `ENV_API_KEY`, `ENV_API_BASE`
- Prefix `DEFAULT_` for defaults: `DEFAULT_API_BASE`, `DEFAULT_ENDPOINT`, `DEFAULT_THINKING_LEVEL_MAP`

### Files
- kebab-case: `config-store.ts`, `error-handler.ts`
- One module per file, one concern per module

## Error Handling

### Pattern: Never Throw, Return Undefined
- I/O functions return `undefined` on failure, never throw (except `login()` for empty key)
- Callers check for `undefined` and fall back gracefully

### Pattern: Try/Catch at Boundaries
- All network and file I/O is wrapped in try/catch
- ENOENT silently suppressed (credential files may not exist)
- Other errors logged with `console.warn` prefix `[agy]`
